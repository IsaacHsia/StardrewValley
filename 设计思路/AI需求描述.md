我现在想要做一个首辅模拟器（文字冒险）和杀戮尖塔2（卡牌肉鸽）结合的游戏项目，他的大概规则是这样的：
1. 事件
	与杀戮尖塔的房间类似，区别在于每一个事件对玩家来说相当于一个回合，所以玩家在每一个事件房都可以执行正常的打牌，事件分为以下几种
		1. 普通房：与杀戮尖塔2问号房相同，包装模式为玩家作为首相收到大臣的奏折，玩家需要在2-3个选项内做出选择并获得对应效果，UI上显示在界面左下区域，区域内显示事件图片、事件名称、事件描述、事件选项以及选项效果，需要注意的是游戏选项需要玩家的属性或当前状态满足一定条件才能选；
		2. 战斗房：包装为急报，出现在界面右上区域，区域内会显示事件名称、事件描述、事件下回合效果（参考杀戮尖塔的敌人意图）和事件进度（敌人HP）；
		3. Boss房/精英怪房：与战斗房的区别在于，敌人包装会是国家级别的敌人，例如敌国、天灾等，考验玩家对于国家属性的培养（国库、军力）
		4. 休沐：玩家可以选择休息（恢复圣眷HP）或是宴请（消耗HP获得锦囊卡牌），休息和杀戮尖塔2的篝火房间一样，可以选择恢复生命或者升级锦囊，宴请和杀戮尖塔2的商人一样，可以选择购买卡牌、道具或者删除卡牌
		5. 回合结束逻辑：在战斗房，打完牌点“结束回合”，敌人攻击；在普通房，玩家点击了某个选项后算作回合结束，玩家出牌是为了通过提升当前回合的状态或者提供足够进度，以此满足选项要求
		6. 普通房没有“敌人血量”，但玩家可以通过打出带有“增加权势/机敏”的牌，**临时满足**某个灰色高收益选项的解锁条件。打牌是手段，点击奏折选项是这个房间特有的“结束回合并结算”的方式。
2. 卡牌（锦囊）
	1. 锦囊类型
		1. 手牌
			1. 效果：参考杀戮尖塔，攻击=行动，技能=策略，能力=身份
		2. 身份牌：参考杀戮尖塔的能力牌，放置方式类似小丑牌中的小丑，玩家有放置上限；当放置区满时，如果玩家又打出一张身份牌，允许玩家选择覆盖/销毁一张已有的身份牌；销毁身份牌不需要付出代价；
	2. 锦囊结算：同杀戮尖塔2的伤卡牌效果结算逻辑
	3. 锦囊牌堆：同杀戮尖塔2的牌堆运作逻辑
3. 选择路线
		游戏中，参考杀戮尖塔2的地图路线生成逻辑，在玩家选择开始游戏时就生成，包装模式为古代钦天监给出历法方案，玩家作为首辅可以选择历法并在游戏中查看历法选择下一回合去哪个事件
4. 玩家属性
	1. 全局属性：
	圣眷：玩家的HP，归0是游戏结束，进入死亡结算
	国库：国家属性，初始不为0，卡牌特殊消耗资源；政令每回合会自动回满。但如果一张卡牌的特殊消耗是“国库 -5”，这 5 点国库是**永久扣除**的
	军力：国家属性，初始不为0，BOSS战斗中玩家造成的伤害和受到的伤害都需要先加上和减去军力数值，军力在 Boss 战中作为增减伤系数，它是按绝对值加成（例如军力 10，每张攻击牌额外 +10 进度）
	
	
	## 战斗相关属性对应

| 杀戮尖塔术语        | 首辅模拟器对应            |
| ------------- | ------------------ |
| 力量（Strength）  | **权势** — 朝中影响力     |
| 敏捷（Dexterity） | **机敏** — 政治自保能力    |
| 格挡（Block）     | **根基** — 政治地位稳固度   |
| 能量（Energy）    | **政令** — 每回合可用的行动力 |
| 消耗（Exhaust）   | **焚毁** — 销毁证据/用后即弃 |
| 伤害            | 进度                 |
这套设计的核心思路是：**把卡牌战斗的攻防映射为朝堂上的政治攻防**——攻击即弹劾/清洗，防御即自保/庇护，增益即结党/揽权，诅咒即流言/猜忌。玩家打出的每一张牌都像在进行一场朝堂博弈。

5. UI显示
	1. 手牌区：在界面右下区域，主要显示玩家当前有的手牌，其他包括抽排堆、抽排堆数量、弃牌堆、弃牌堆数量、消耗堆、消耗堆数量，也包括回合结束按钮；
	2. 放置区：在手牌区上方，显示玩家当前放置的身份牌；身份牌槽位区支持拖拽替换；
	3. 玩家状态区：玩家的血量（圣眷）、血量条（参考杀戮尖塔2的显示形式）、权势、机敏、根基以及获得的buff，一起作为玩家状态放在手牌区的下方显示；
	4. 事件区：界面右上为普通房事件显示，上部分为事件描述，下部分为事件选项；
	5. 战斗区：界面左半部分区域为战斗区，显示战斗房、精英房和boss房事件，最上方显示战斗事件描述，然后显示敌人下回合的意图，意图下方为事件血量；存在多个敌人同时出现的可能，玩家通过拖动卡牌指向不同的事件来选择目标，先出现的目标在上面，后出现的在下面
	6. 装备区：手牌区下方、界面底端是装备区，参考杀戮尖塔的遗物
6. 配置结构
	1. 事件表Events_config：事件ID、事件名称、事件描述、图片路径、事件标签、选项1文本、选项1效果（引用效果表）、选项2文本、选项2效果、选项3文本、选项3效果
	2. 卡牌表Cards_config（锦囊效果使用Lua脚本和表格一起实现）：锦囊id、锦囊名称、锦囊描述、锦囊类型（0=手牌，1=锦囊）、稀有度（0=普通，1=稀有，2=史诗）、目标（0=自身，1=敌方单体，2=敌方全体，3=敌方随机），政令消耗，流派tag、使用lua脚本名称，自带buff（引用buff表）、升级buff（引用buff表）
	3. 卡牌lua脚本：实现一些较为复杂的卡牌逻辑，其中至少需要包括：卡牌的Buff触发条件
	4. Buff表Buff_Config：BuffID、Buff类型、Buff作用目标、修改属性、运算规则、具体数值、持续回合（-1=永久）、
	5. 敌人表Enemies_config（锦囊效果使用Lua脚本和表格一起实现）：敌人ID、敌人名称、敌人事件出现时描述、敌人HP、是否Boss、敌人使用脚本名称、敌人所在；敌人池页签：敌人池id、敌人池名称、敌人池包含敌人
	6. 地图表Map_config：配置ID、第几个区域、楼层总数、战斗房间权重、精英房权重、休息房权重、精英最早出现层数、精英连续出现上线数量、连续战斗出现数量、分支路径数量、普通敌人池（引用敌人池id）、精英敌人池、普通敌人奖励池、精英敌人奖励池、每层属性提升
	7. 奖励表Rewards_config：奖励ID、奖励名称、奖励内容（配置形式为物品:数量，数量波动用[表示上下限]，例如1-150金币就是coin:1,[1,150]，多个奖励内容之间用;分割）
