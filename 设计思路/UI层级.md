## 三、game_screen.tscn — 核心游戏场景

这是最复杂的场景，包含 6 个面板，运行时根据状态显示/隐藏。

### 整体布局示意

### 3.1 TopHUD — 顶部状态栏

|节点名|类型|显示内容|颜色|作用|
|---|---|---|---|---|
|HUDBg|ColorRect|—|`(0.04,0.04,0.08,0.85)`|HUD 半透明背景|
|FavorLabel|Label|"圣眷: 80/80"|🔴 红 `(0.9,0.3,0.3)`|当前/最大圣眷(HP)|
|DecreeLabel|Label|"政令: 3/3"|🔵 蓝 `(0.3,0.7,0.9)`|当前/最大政令(能量)|
|TreasuryLabel|Label|"国库: 50"|🟡 黄 `(0.9,0.8,0.3)`|国库数值|
|MilitaryLabel|Label|"军力: 30"|🟢 绿 `(0.6,0.9,0.4)`|军力数值|
|GoldLabel|Label|"金币: 100"|🟡 金 `(0.9,0.75,0.3)`|金币数量|
|StatusLabel|Label|"权势+2 机敏+1"|🟣 紫 `(0.8,0.6,1.0)`|当前战斗增益摘要|

### 3.2 CombatPanel — 战斗面板（急报）

**位置**: 右半区域 (50%宽 ~ 右边缘)，顶部45px ~ 55%高度 **显示时机**: 战斗房(combat)

|节点名|类型|显示内容|作用|
|---|---|---|---|
|TurnLabel|Label|"回合 1"|当前战斗回合数|
|EnemyNameLabel|Label|"北蛮大举南侵"|敌人名称，红色字号22|
|EnemyDescLabel|Label|"北方游牧联盟集结..."|敌人描述，灰色自动换行|
|EnemyHPBar|ProgressBar|血条|敌人HP进度条(不显示百分比)|
|EnemyHPLabel|Label|"120 / 150"|敌人当前/最大HP数值|
|IntentLabel|Label|"意图: 铁骑冲锋 — 造成18点进度"|敌人下回合行动意图，橙色|

### 3.3 EventPanel — 事件面板（奏折）

**位置**: 左侧 (20px ~ 500px宽)，顶部45px ~ 75%高度 **显示时机**: 事件房(event)

|节点名|类型|显示内容|作用|
|---|---|---|---|
|EventNameLabel|Label|"税制改革奏折"|事件标题，金色字号24|
|EventDescLabel|Label|"户部尚书上奏..."|事件描述，浅灰自动换行|
|Sep|HSeparator|—|分隔线|
|OptionsContainer|VBoxContainer|(动态生成选项按钮)|事件选项容器|

**OptionsContainer 内的动态按钮：**

|状态|样式|示例|
|---|---|---|
|可选|正常按钮 350×50|"推行一条鞭法\n(国库+15，失去5圣眷)"|
|条件不足|灰化禁用|"加征商税\n(国库+10) [条件不足]"|

### 3.4 RestPanel — 休沐面板

**位置**: 屏幕居中弹窗 (400×220) **显示时机**: 休沐房(rest)

|节点名|类型|显示内容|作用|
|---|---|---|---|
|Title|Label|"休沐"|标题，绿色字号28|
|RestBtn|Button|"休息 (恢复30%圣眷)"|恢复圣眷，返回地图|
|FeastBtn|Button|"宴请 (消耗5圣眷, 获得随机锦囊)"|消耗HP获得随机卡牌|

### 3.5 RewardPanel — 奖励选择面板

**位置**: 屏幕居中弹窗 (740×180) **显示时机**: 战斗胜利后

|节点名|类型|显示内容|作用|
|---|---|---|---|
|Title|Label|"选择一张锦囊加入牌组"|标题，金色|
|RewardContainer|HBoxContainer|(动态生成卡牌按钮)|奖励卡牌容器|

**RewardContainer 内的动态按钮：**

|元素|尺寸|示例|
|---|---|---|
|奖励卡按钮|180×100|"[1] 密谋布局\n获得2点权势。"|
|跳过按钮|100×100|"跳过"|

### 3.6 HandManager — 手牌管理器

**位置**: 底部居中，距底边45~170px **显示时机**: 战斗房和事件房

|元素|说明|
|---|---|
|内部无预置子节点|运行时动态创建 CardUI 实例|
|CardUI × N|扇形排列的手牌卡牌组件|

**每张 CardUI 的卡面结构：**

```
┌──────────────────────┐  ← 外发光 (稀有度颜色)
│ ◆1    [焚][固]       │  ← 费用徽章 + 关键词角标
│ ┌──────────────────┐ │
│ │   暗色插画区域     │ │  ← 80px高 (预留纹理)
│ └──────────────────┘ │
│ ── 弹劾奏章 ──────── │  ← 名称 (升级后变绿+加号)
│ ──────────────────── │
│  造成6点进度。        │  ← 效果描述
│       手牌           │  ← 类型标签
└──────────────────────┘
   140×195px
```

|CardUI 子元素|作用|
|---|---|
|_glow_rect|外发光边框，颜色=稀有度(普通铜/罕见蓝/稀有金)|
|_frame_rect|卡框背景，颜色=稀有度|
|_art_rect|插画占位区 128×80|
|_cost_bg + _cost_label|左上角费用徽章 "1" (可出=金色，不可出=暗红)|
|_name_label|卡牌名称 (升级后绿色+"号)|
|_desc_label|效果描述文本|
|_type_label|底部类型标签 "手牌"(暖橙) / "放置"(青绿)|
|_keyword_container|关键词角标 "焚""固""留""虚"|

### 3.7 BottomBar — 底部操作栏

**位置**: 屏幕最底部，全宽，42px高，z_index=300（始终最顶层）

|节点名|类型|显示内容|颜色|作用|
|---|---|---|---|---|
|BottomBg|ColorRect|—|不透明深色|遮挡手牌溢出|
|DrawPileLabel|Label|"抽牌堆: 5"|🔵 蓝 `(0.4,0.65,0.85)`|抽牌堆剩余数量|
|EndTurnBtn|Button|"结束回合"|—|结束当前回合 140×36 字号18|
|PileLabel|Label|"弃牌堆: 0 \| 焚毁: 0"|🟤 棕 `(0.6,0.5,0.4)`|弃牌堆和焚毁堆数量|

## 五、event_editor.tscn — 数据编辑器

| 节点名          | 类型           | 显示内容                 | 作用           |
